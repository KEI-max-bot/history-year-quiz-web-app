# UI/UX 設計仕様

## 設計方針

Apple風のUI/UXを目指す。シンプルで直感的、美しいデザインを基本とする。

## 原則

### 1. ミニマリズム
- 必要最小限のUI要素のみ表示
- 余白を活用し、詰め込まない
- 画面ごとに1つの主要アクションに集中

### 2. 視覚的ヒエラルキー
- 重要な要素ほど大きく、目立つ位置に配置
- フォントサイズ・太さで階層を表現
- 背景とのコントラストで注目を誘導

### 3. 直感的な操作
- ボタンは押せそうに見える（アフォーダンス）
- タップ・クリックに即座に反応（フィードバック）
- 間違えても元に戻せる安心感

### 4. 一貫性
- 色、角丸、影、間隔のルールを統一
- 同じ見た目は同じ動作をする

### 5. アニメーション
- 自然な動き（物理法則に基づく）
- 状態変化を伝える目的のある動き
- 適度な速度（200-400ms程度）

## 実装ガイドライン

### 通知・フィードバック
| 避けるべき | 推奨 |
|------------|------|
| alert() | トースト通知、インライン表示 |
| confirm() | モーダルダイアログ（カスタムUI） |
| 即座の状態変化 | フェード・スライドアニメーション |

### 色の使い方
- **アクセントカラー**: 主要アクション（1色に絞る）
- **成功**: 緑系（正解、完了）
- **エラー**: 赤系（不正解、警告）
- **ニュートラル**: グレー系（補助的な要素）

### インタラクション
- ホバー時: 軽い色変化または影の変化
- クリック時: 押し込み感（scale: 0.98程度）
- フォーカス時: 明確なアウトライン

### レスポンシブ
- モバイルファースト
- タップ領域は最低44x44px
- 横幅に応じたレイアウト調整

## 現状と改善方針

| 項目 | 現状 | 改善方向 |
|------|------|----------|
| 通知 | alert使用 | トースト/インライン化 |
| 正解/不正解 | 一瞬の色変化 | より明確なフィードバック |
| ボタン配置 | 複数並列 | 主要アクションを強調 |
| 余白 | 狭め | 余裕を持たせる |
| アニメーション | なし | 状態遷移に追加 |
