# Data Specification

## Purpose
世界史の年号データを管理し、クイズや一覧表示に提供する。

## Requirements

### Requirement: データ構造
システムは年号データを構造化された形式で保持しなければならない。

#### Scenario: 基本データ形式
- GIVEN 年号データが定義される
- WHEN データを参照する
- THEN 各データは以下のプロパティを持つ
  - year: 年号（整数、紀元前は負の数）
  - event: 出来事名（文字列）
  - category: カテゴリ名（文字列）

#### Scenario: 拡張データ形式
- GIVEN 年号データに追加情報がある
- WHEN データを参照する
- THEN オプションで以下のプロパティを持てる
  - img: 画像URL（文字列）
  - alt: 画像の代替テキスト（文字列）
  - note: 詳細な解説（文字列）

---

### Requirement: カテゴリ分類
システムは年号データをカテゴリ別に分類しなければならない。

#### Scenario: カテゴリ一覧
- GIVEN カテゴリが定義されている
- WHEN カテゴリを取得する
- THEN 以下のカテゴリが存在する
  - 古代オリエント・地中海世界
  - 東・南アジア1（〜999年）
  - 東・南アジア2（1000年〜1799年）
  - イスラーム世界（〜1799年）
  - 中世ヨーロッパ（477年〜1476年）
  - 近代ヨーロッパ
  - 現代世界史1（1800年〜1899年）
  - 現代世界史2（1900年〜1945年）
  - 現代世界史3（1946年〜）
  - その他

---

### Requirement: データの追加
システムはユーザーが新しい年号データを追加できるようにしなければならない。

#### Scenario: 入力モードでデータ追加
- GIVEN ユーザーが入力モードにいる
- WHEN 年号、出来事名、カテゴリを入力して「追加」ボタンを押す
- THEN システムは新しいデータをリストに追加する

#### Scenario: バリデーション
- GIVEN ユーザーがデータを追加しようとする
- WHEN 必須項目（年号、出来事名）が空である
- THEN システムはエラーを表示し、追加を拒否する

---

### Requirement: データの表示
システムは年号データを一覧表示できなければならない。

#### Scenario: 世紀別表示
- GIVEN ユーザーが「世紀別」タブを選択している
- WHEN 年号一覧が表示される
- THEN データは世紀ごとにグループ化されて表示される

#### Scenario: カテゴリ別表示
- GIVEN ユーザーが「カテゴリ別」タブを選択している
- WHEN 年号一覧が表示される
- THEN データはカテゴリごとにグループ化されて表示される
